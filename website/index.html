Quick start

Implement a handler on either side and connect it to a transport. Every method is typed against the ACP schema.

Agent

dart run example/basic_agent.dart 
import 'package:acp/agent.dart';
import 'package:acp/schema.dart';
import 'package:acp/transport.dart';

final class EchoAgent extends AgentHandler {
  final AgentSideConnection conn;
  EchoAgent(this.conn);

  @override
  Future<InitializeResponse> initialize(
    InitializeRequest req, {
    required AcpCancellationToken cancelToken,
  }) async =>
      const InitializeResponse(protocolVersion: 1);

  @override
  Future<PromptResponse> prompt(
    PromptRequest req, {
    required AcpCancellationToken cancelToken,
  }) async {
    await conn.notifySessionUpdate(
      req.sessionId,
      AgentMessageChunk(
        content: const TextContent(
          text: 'Hello from Dart',
        ).toJson(),
      ),
    );
    return const PromptResponse(stopReason: 'end_turn');
  }
}

void main() {
  final transport = StdioTransport()..start();
  AgentSideConnection(transport, handlerFactory: EchoAgent.new);
}

Client

dart run example/basic_client.dart 
import 'package:acp/client.dart';
import 'package:acp/schema.dart';
import 'package:acp/transport.dart';

final class Printer extends ClientHandler {
  @override
  void onSessionUpdate(String id, SessionUpdate update) {
    if (update case AgentMessageChunk(:final content)) {
      print(content['text']);
    }
  }
}

Future<void> main() async {
  final transport = await StdioProcessTransport.start(
    'dart', ['run', 'example/basic_agent.dart'],
  );

  final client = ClientSideConnection(
    transport,
    handler: Printer(),
    clientCapabilities: const ClientCapabilities(
      fs: FileSystemCapability(
        readTextFile: true,
        writeTextFile: true,
      ),
      terminal: true,
    ),
  );

  await client.sendInitialize(protocolVersion: 1);
  final s = await client.sendNewSession(cwd: '.');
  await client.sendPrompt(
    sessionId: s.sessionId,
    prompt: const [TextContent(text: 'hi')],
  );
}

Common patterns

Streaming, cancellation, and capability checks — the pieces you write once and forget about.

lib/agent/streaming.dart 
// Agent side: stream tokens, plans, and tool calls as they happen.
@override
Future<PromptResponse> prompt(
  PromptRequest req, {
  required AcpCancellationToken cancelToken,
}) async {
  await for (final token in generate(req.prompt)) {
    if (cancelToken.isCancelled) break;
    await conn.notifySessionUpdate(
      req.sessionId,
      AgentMessageChunk(content: TextContent(text: token).toJson()),
    );
  }
  return const PromptResponse(stopReason: 'end_turn');
}

// Client side: destructure updates with pattern matching.
@override
void onSessionUpdate(String sessionId, SessionUpdate update) {
  switch (update) {
    AgentMessageChunk(:final content) => emit(content['text']),
    PlanUpdate(:final entries) => renderPlan(entries),
    ToolCallUpdate(:final status) => showStatus(status),
    _ => null,
  };
}

// Every handler receives an AcpCancellationToken.
@override
Future<PromptResponse> prompt(
  PromptRequest req, {
  required AcpCancellationToken cancelToken,
}) async {
  final sub = cancelToken.onCancel.listen((_) {
    log.info('client cancelled ${req.sessionId}');
  });

  try {
    final result = await runModel(
      req.prompt,
      cancelWhen: cancelToken.future,
    );
    return PromptResponse(stopReason: result.reason);
  } on AcpCancelled {
    return const PromptResponse(stopReason: 'cancelled');
  } finally {
    await sub.cancel();
  }
}

// Declare capabilities once. The runtime enforces them on both sides.
final client = ClientSideConnection(
  transport,
  handler: MyClient(),
  clientCapabilities: const ClientCapabilities(
    fs: FileSystemCapability(
      readTextFile: true,
      writeTextFile: false,
    ),
    terminal: true,
  ),
  capabilityEnforcement: CapabilityEnforcement.strict,
);

// Agent side: respond to initialize with what you support.
@override
Future<InitializeResponse> initialize(...) async {
  return const InitializeResponse(
    protocolVersion: 1,
    agentCapabilities: AgentCapabilities(
      sessionCapabilities: SessionCapabilities(list: {'stable': true}),
      promptCapabilities: PromptCapabilities(image: true, embeddedContext: true),
    ),
  );
}

What's included

The package targets the ACP v0.12.0 schema and wraps the mechanics you'd otherwise rewrite for every agent and every editor.

Typed schema models

Generated request, response, notification, and content models. Sealed unions with extensionData for forward compatibility.

Full JSON-RPC runtime

Requests, responses, notifications, batching, timeouts, and connection state — plus $/cancel_request plumbing.

Capability enforcement

Calls blocked when the other side never advertised the capability. Pick strict or permissive for integration work.

Cooperative cancellation

Every handler receives an AcpCancellationToken. Unwind gracefully without leaking streams or orphaning work.

Streaming session updates

Push AgentMessageChunk, plan updates, tool calls, and terminal events as they happen with a TerminalHandle helper.

Extension methods

Client-to-agent and agent-to-client custom methods with the _ prefix convention, round-tripped through _meta.

Transports

Start with stdio for local subprocess agents. Switch to HTTP or WebSocket once the agent and client are separated by a network boundary. Implement AcpTransport for anything else.

Class Description Side
StdioTransport Agent-side NDJSON over stdin/stdout. agent
StdioProcessTransport Client-side subprocess spawning and management. client
HttpSseTransport HTTP POST plus Server-Sent Events. both
StreamableHttpTransport Streamable HTTP transport, chunked both ways. both
WebSocketTransport VM WebSocket text frames. vm
BrowserWebSocketTransport Browser-side WebSocket text frames. web
ReconnectingTransport Reconnect wrapper for reconnectable transports. wrapper
AcpTransport Interface for implementing custom transports. byo
